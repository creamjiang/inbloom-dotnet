Sample Application

The sample application is a simple "hello world" type application designed to demonstrate OAuth authentication and .NET Library usage.

Source

Source for the sample application is included in the distribution under the inBloom-Test-webapp directory. The sample application is provided as an inBloomApiClassLibrary project. To build the application from this source, Open this application in Microsoft Visual Studio and run the sample application.

Prerequisites

Before running the sample application, your application must have access to data in an inBloom Data Store. Use other sections of inBloom developer documentation to determine how to set up your development environment, including adding data to the data store.

sli.sample.apiUrl

URL of the inBloom REST API the application's using. Please ensure the URL matches the desired environment (Sandbox or Production). (Example: https://api.sandbox.example.org/)

sli.sample.callbackUrl

OAuth callback URL for the sample application. inBloom redirects to this URL after a user authenticates with its identity provider. This must match the redirect_url configured when requesting a new application. (Example: https://myapp.example.com/callback)

sli.sample.clientId

Unique string identifying this application, created during the application registration process with the target inBloom environment.

sli.sample.clientSecret

Secret used to authenticate this application, created during the application registration process with the target inBloom environment. Care should be taken to ensure this value is secure.*


* Production apps should ensure this value is encrypted or otherwise locked down.

The sample application searches for the sample.properties file by looking up the value of a .NET application configuration called app.config.

Running using IIS

You can run the sample application with IIS also. Before that the application should configured to test this sample application.

Using the Sample Application

Once the sample application is running, access it using the following steps:

  1. Visit the sample application's URL in your browser. (Example: http://example.com/sample/students)

  2. The sample application will redirect you to the inBloom for authentication. Choose the appropriate realm and authenticate with the identity provider.

  3. After authentication, inBloom redirects you back to the sample application following authentication.

  4. The sample application will display your name, a list of roles your logged in user currently has, and any students the logged in user can see. If you do not see any students, students may not exist within the educational organizations the application is approved to see. The current user may not have the appropriate context to see any students (for example, a teacher may not have any students associated with his or her sections).
