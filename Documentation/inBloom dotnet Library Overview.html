inBloom .NET Library Overview

Upeo provides the inBloom .NET Library as a convenient option for .NET programmers when developing applications that use the inBloom REST API. This documentation covers setting up and using the .NET library.

Before you can access the library, you'll need to do the following:

  • Have a developer account with the inBloom as described in Sign Up and Log In.

  • Register your application so that you can use in your development environment. See Registering Your Application in the Sandbox for application registration in a Sandbox development environment, and see Registering Your Application in Production for Production application registration.

Note the following values configured or assigned during the application registration process. You'll need these when using the library:

  • apiURL - The base URL of where the inBloom REST API is hosted. The URLs of all REST API requests will be relative to this URL. This will be in the form $BASE_URL/api/rest/v1.1/.

  • clientId - A unique ID of the client assigned during application registration.

  • clientSecret - A cryptographically secure secret shared between the client and the REST API service, assigned alongside the client ID during application registration.

  • callbackURL - If you're developing a web application, this is the URL where you're currently hosting the application. During development, this is likely a persistent test server location that's running your application.
